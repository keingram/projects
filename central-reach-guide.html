Navigating the Tree

Every client's goals live in a Learning Tree — a folder structure that holds LTGs, STGs, targets, and supporting documents.

Getting to a client's tree

1
Open the ☰ → Clinical menu.
2
Search for the client by name.
3
Click on their Learning Tree (listed under the Learning Trees tab).

What you'll see

The tree has two panels: a folder tree on the left and a content/graph area on the right. Click any item in the left tree to display it on the right. You can toggle the tree panel with "Hide Tree View" at the top.

Tree dashboard (home view)

When you first open a tree, you see the dashboard with two donut charts — Long Term Goals Mastered and Short Term Goals Mastered — plus a count of Goals Met vs. Goals In Progress. This gives a quick snapshot before drilling into individual targets.

For a quick audit, click Tree Options → Graph Analysis from any tree to jump straight to the full-screen graph view where you can filter by status (Regression, No Progress, In Progress, Met).

Sidebar Icons Guide

When you click a target in the tree, a column of icons appears on the right edge of the graph. Here's each one:

☰
Description — View the target's written instructions: SD, prompting procedure, tech statement template, examples.
💬
Comments — Read or leave notes for the team about this target.
✏️
Edit — Open the Branch Details editor to change the title or description. Saves immediately when you click "Done Editing."
＋
Add Branch — Create a new child branch under this item (e.g., add a new target under an STG).
🕐
Data History — See a table of every data point, phase change, and deletion for this target. Great for auditing.
⊕
Add Data — Manually enter a new data point.
⊙
View Goal — See and update goal details: domain, type, dates, and status.
If you only see Description and Comments, you probably don't have edit rights to that tree. The tree creator or an admin can grant access.

Bottom Dot Navigation

At the very bottom of the graph view, a row of blue dots with left/right arrows lets you jump between targets without clicking through the tree.

Left arrow (←) = previous target. Right arrow (→) = next target. Click any dot to jump directly — the filled/larger dot is where you are now. Hover over a dot to see the target name in a tooltip.

This is the fastest way to review multiple targets in sequence during supervision.

The Inline Graph

When you click a target in the tree, the graph appears on the right. This is the view you'll use most.

A typical target-level graph showing baseline data, intervention data climbing toward the goal line.

What's on the graph

Y-axis = what you're measuring (usually Percent Correct, but could be frequency/rate/duration depending on the target setup). X-axis = session dates. Each dot is one session's data, connected by lines.

The legend at the bottom shows: Baseline, Intervention, Generalization, Maintenance, Event, Goal, Phase Change. Different phases get different colored dots so you can see transitions.

The goal line (red dashed horizontal line) is the mastery criterion. The phase change line (vertical dashed line) marks where one phase ended and the next began.

In the top-right corner: "Target Goal — Initiated on [date]" and the status badge (In Progress, Met, etc.).

LTG vs. Target Graphs

The graph looks different depending on whether you clicked an LTG or a Target.

Different views for different purposes. If you're looking at a flat "Goal Count" graph, you're at the LTG level — drill into a target.

The "Cumulative Weekly" dropdown

This only appears on the LTG-level graph. It controls how mastered-goal data is grouped:

OptionShows
Cumulative Weekly / Monthly / QuarterlyRunning total of goals met, grouped by time period
Percent Weekly / Monthly / QuarterlyPercentage of goals met out of total, by time period
For progress reports, use Cumulative Monthly or Percent Monthly to show payors a clear picture of mastery progress across the authorization period.

Above every target graph, you'll see zoom buttons and date pickers:

The From / To date pickers let you set a custom range — useful for matching authorization periods exactly.

Graph looks empty? Check the date range first. If the From/To is too narrow, data points outside that window won't show. Click "All" to see everything.

Exporting & Printing

Click the hamburger menu (≡) in the top-right corner of any graph to see export options:

🖨
Print chart — sends to your printer.
📷
Download PNG / JPEG / SVG — saves as an image. You get two sub-options: "WITH blue navigation bar" and "WITHOUT blue navigation bar."
📊
Download CSV — exports raw data as a spreadsheet file.
For progress reports, use "Download PNG — WITHOUT blue navigation bar" for a clean graph image you can paste directly into Word or a PDF report.

Graph Analysis Page

A separate full-screen charting tool. Get there via Tree Options → Graph Analysis (or Branch Options → Graph Analysis).

Chart types

TypeWhat It ShowsWhen to Use
ScatterplotData points by time of day (Y = time, X = date)Check if time-of-day affects performance
Advanced AnalysisData grouped by time intervals (configurable)Compare across days of week, hours, etc.
Learning Opps Per DateTrial counts per sessionCheck if enough opportunities are being run
ABC Bar / Line / PieAntecedent-Behavior-Consequence dataBehavior function analysis

Status filters (left sidebar)

The left sidebar on this page has clickable status filters: New, Regression, No Progress, In Progress, Met. Click Regression and No Progress to quickly see which targets need attention.

Scatterplot controls

Daily Time Window (From/To hours) narrows the view to specific hours. Y-axis Interval sets the time scale (Half hour, Hour, Two hours). Show gridlines and Show data labels toggle visual aids.

Advanced Analysis controls

Smaller Interval = unit of measurement (15 min, 30 min, hour, day). Larger Interval = grouping period (hour, day, week, month). Try Day + Week to see which days perform best, or Hour + Day for time-of-day patterns.

Trendlines

The Linear Regression line (visible in the LTG graph legend and the Graph Analysis page) is a straight line drawn through your data that shows the overall direction, smoothing out day-to-day noise.

Angled up = progress is happening. Flat = no change, may need a procedure adjustment. Angled down = performance declining, investigate.

Trendlines need at least 5+ data points in a phase to be reliable. Don't read too much into a trendline based on 2-3 sessions.

The inline target graph doesn't show a trendline by default — you see it in the LTG-level graph (as "Linear Regression" in the legend) and in the Graph Analysis → Scatterplot view.

Data History Table

Click the Data History (🕐) icon to see a table of every data entry for a target.

ColumnWhat It Shows
UserWho entered the data (e.g., "L. Jones" = RBT, "P. Momtsios" = BCBA)
TypeData point, Phase Change (colored badge), or Event
CommentNotes attached (e.g., "Intervention")
DataThe actual score/value
Learning Opp.How many trials were given that session
DateWhen collected

Deleted entries show with a strikethrough and display who deleted them, when, and the reason. This creates an audit trail.

Summary stats at the bottom

Total Learning Opportunities = sum of all trials. Total Data Points = number of sessions. Learning Opportunities Per Data Point = average trials per session. If this is very low, the RBT may not be running enough trials.

Phase Change Lines

A phase change line is the vertical dashed line on the graph separating phases (e.g., Baseline → Intervention). The line goes between the last data point of the old phase and the first data point of the new phase.

Adding a phase change

1
Click the target so its graph is showing.
2
Click Data History (🕐) in the sidebar.
3
Use the + button to add a new phase change.
4
Select the phase type (Baseline, Intervention, Generalization, Maintenance) and the correct date.

If you placed it at the wrong date

Open Data History, find the phase change entry, delete it (you'll type a reason like "incorrect date"), then add a new one at the right date. The deleted entry stays visible in Data History with a strikethrough as an audit trail, but it won't affect the graph.

Fixing a Wrong Data Point

1
Open Data History (🕐).
2
Find the incorrect data entry.
3
Delete it with a reason (e.g., "incorrect score — should be 4/5 not 2/5").
4
Click Add Data (⊕) to enter the correct value at the correct date.
CR asks for a reason when you delete — something short like "wrong score entered" is fine and helps when reviewing the history later.

Viewing & Updating Goal Status

Click View Goal (⊙) in the sidebar to open the Goal Details panel.

FieldWhat It Means
DomainSkill area (communication, social, adaptive, etc.)
TypeUsually "Target" for STG items
ResponsibilityWho's responsible
Date InitiatedWhen data collection started
In Progress DateWhen active intervention began

Status options

Not Active → In Progress → On Hold / Met / Discontinued

Click the status button you want, then click Update.

Update saves to this target only. Update All applies the change to every target in the branch — so double-check which one you're clicking.

Editing Goal Instructions

To change a target's SD, prompting procedure, tech statement, or examples:

1
Click the target in the tree.
2
Click Edit (✏️) in the sidebar.
3
The Branch Details panel opens with two fields: Branch Title and Branch Description (rich text editor — supports bold, underline, lists, tables, images).
4
Make your changes.
5
Click "Done Editing" to save, or "Cancel" to discard.
"Done Editing" saves immediately — if you're making big changes, it's worth copying the current text somewhere first just in case.

Adding New Targets

1
Click the parent branch (the STG or LTG you want to add under).
2
Click Add Branch (＋) in the sidebar.
3
Enter the Branch Title and Description.
4
Click Done Editing.
5
Then open View Goal (⊙) to set the Domain, dates, and mark as "In Progress."
If the new target is similar to an existing one, use Branch Options → Copy Branch on the existing target, then edit the copy — faster than starting from scratch.

Archiving & Discontinuing

Three ways to handle goals that are done or no longer needed:

Mark as "Met"

→View Goal (⊙) → Met → Update. Target stays visible in the tree with a "Met" badge. Data preserved.

Mark as "Discontinued"

→View Goal (⊙) → Discontinued → Update. Goal removed from the active plan but stays in the tree.

Archive the Branch

→Branch Options → Archive Branch. Hides from the tree entirely. Restore later from Tree Options → Archived Branches.

Branch Options Menu

Click "Branch Options" (top-right of the tree view, or the three-dot menu) to access:

OptionWhat It Does
Copy BranchDuplicate this branch to the same or another client's tree
Bulk Copy BranchCopy to multiple clients at once
Bulk Phase ChangesChange the phase on all targets under this branch simultaneously
Bulk EventsAdd an event marker across all targets (e.g., illness, vacation)
Change to DraftMark as draft — shouldn't be used for data yet
Lock BranchPrevent any edits (finalized goals)
Archive BranchHide from active tree view
Bulk Phase Changes affects everything below the level you're on — at the LTG level, that means every target under it. Just check you're at the right level first.

Also accessible from this menu: Graph Analysis, ABC History, Printable Datasheet, and Create a Session.

The Files Section

All uploaded documents (PDFs, forms, assessments) live here: ☰ → Files.

The Files page: folder tree on the left, searchable file list on the right, with date and type filters.

The page has two main tabs: CURRENT (recent files) and ARCHIVED (older documents moved automatically). Each file has a Preview button to view without downloading.

Searching & Filtering Files

🔍
Search bar — type part of a file name to filter instantly. Fastest way to find something specific.
📅
Date filter — narrow to documents uploaded within a specific date range.
📁
Folder tree — click a folder (Client Documentation, etc.) to filter by category.
🏷
LABELS / DOCUMENT TYPE tabs — filter by tags or form template type.
🔒
Classified toggle — show only classified or unclassified documents.
✅
ALL vs AUTHORIZED — "ALL" shows everything; "AUTHORIZED" shows only files linked to authorized services.
Can't find a document? 1. Click "CLEAR FILTERS" first. 2. Make sure you're on "ALL," not "AUTHORIZED." 3. Check the "ARCHIVED" tab. 4. Try searching with just part of the filename.

Documents in the Learning Tree

Many client documents also live inside the Learning Tree as branches. When you open a client's tree, you'll see folders like:

Client Face Page · Session Tips · Coordination of Care Provider List · Evaluation Information · Client Information & Progress Summary · Background Information · Risk Assessment / Behavior Management · Assessment Information

Each of these is a clickable branch. Some have sub-folders with additional documents. Branches with a 📄 icon next to them have viewable document content attached.

Two places to look for documents. If you can't find something in ☰ → Files, check the client's Learning Tree. Client-specific documents (assessments, behavior plans, session tips) often live as tree branches rather than uploaded files.
